BDBulkSMS.net API — Quick Start

This page documents the available endpoints (plain text & JSON), parameters and examples. Keep your token secret.

Primary Endpoints

Note: NON‑SSL (http) support is available if you need it (e.g. legacy systems). Prefer https when possible.

Generate Token

Token generation link: https://gwb.li/token

Use the returned token in queries: token=YOUR_TOKEN_HERE

Send SMS — Parameters

  • message — Text message body.
  • to — Destination number (single or comma separated).
  • token — Your API token.
  • Method — POST or GET accepted.
Example — curl (GET) 
curl "https://api.bdbulksms.net/api.php?token=YOURTOKEN&to=8801XXXXXXXXX&message=Hello%20from%20BDBulkSMS"
Example — curl (POST, JSON response) 
curl -X POST "https://api.bdbulksms.net/api.php?json" \
  -d "token=YOURTOKEN" \
  -d "to=8801XXXXXXXXX" \
  -d "message=Hello from BDBulkSMS"

আজ্ঞা: আপনি http ব্যবহার করতে পারেন যদি https মেনে নেওয়া না যায় — কিন্তু production এ https ব্যবহার করা উচিত।

Notes / Best Practices

  • Always store tokens securely and rotate them if leaked.
  • Use HTTPS in production for security.
  • For bulk sending, respect rate limits and segment lists to avoid throttling.
  • Test with a single number before large batches.
  • Check expiry and balance regularly via the g_api endpoints.

Example JS (fetch)

fetch("https://api.bdbulksms.net/api.php?json", {
  method: "POST",
  headers: {"Content-Type":"application/x-www-form-urlencoded"},
  body: new URLSearchParams({
    token: "YOURTOKEN",
    to: "8801XXXXXXXXX",
    message: "Hello from BDBulkSMS"
  })
})
.then(r => r.json()).then(console.log).catch(console.error);
